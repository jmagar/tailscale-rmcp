# Repository Rules

- Keep `cli.rs` and `src/mcp/tools.rs` thin.
- Put behavior and policy in `src/app.rs`.
- Put raw HTTP details in `src/tailscale.rs`.
- Preserve the MCP `{ ok, data, error }` envelope.
- Keep destructive operations behind server opt-in and caller confirmation.
- Keep `server.json`, npm `mcpName`, and advertised server info aligned.
- Keep `docs/references/` generated by `scripts/refresh-docs.sh`.
- Do not edit `AGENTS.md` or `GEMINI.md` directly when `CLAUDE.md` exists.
